title: "API authentication"
description: "Authenticating your API requests"
---

Every request to a `/v1/video` endpoint must include a valid API key in the request headers. Requests without a key, or with an invalid key, are rejected immediately — no credits are deducted.

## Request header

Include your API key using either of the following headers:

```http
api-x-key: YOUR_API_KEY
```

```http
x-api-key: YOUR_API_KEY
```

Both headers are accepted and behave identically. The server checks for `api-x-key` first. If that header is absent, it falls back to `x-api-key`.

<Warning>
  Your API key is a secret credential. Never include it in client-side JavaScript, commit it to a repository, or share it publicly. Store it in environment variables or a secrets manager and reference it server-side only.
</Warning>

## Examples

<CodeGroup>

```bash curl
curl -X POST https://your-stophy-host/v1/video/transcript \
  -H "Content-Type: application/json" \
  -H "api-x-key: YOUR_API_KEY" \
  -d '{"videoUrl": "https://www.youtube.com/watch?v=dQw4w9WgXcQ"}'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-stophy-host/v1/video/transcript', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'api-x-key': process.env.STOPHY_API_KEY,
  },
  body: JSON.stringify({
    videoUrl: 'https://www.youtube.com/watch?v=dQw4w9WgXcQ',
  }),
});

const data = await response.json();
```

```python Python (requests)
import os
import requests

response = requests.post(
    'https://your-stophy-host/v1/video/transcript',
    headers={
        'Content-Type': 'application/json',
        'api-x-key': os.environ['STOPHY_API_KEY'],
    },
    json={'videoUrl': 'https://www.youtube.com/watch?v=dQw4w9WgXcQ'},
)

data = response.json()
```

</CodeGroup>

## Error responses

### Missing API key

When neither `api-x-key` nor `x-api-key` is present in the request:

```json
{
  "statusCode": 401,
  "message": "Missing api-x-key header"
}
```

**HTTP status:** `401 Unauthorized`

### Invalid API key

When the key is present but does not match any record in the database:

```json
{
  "statusCode": 401,
  "message": "Invalid API key"
}
```

**HTTP status:** `401 Unauthorized`

### User not found

When the API key is valid but the associated user account no longer exists:

```json
{
  "statusCode": 404,
  "message": "User not found for API key"
}
```

**HTTP status:** `404 Not Found`

This indicates a data consistency issue. Contact support if you receive this error.
