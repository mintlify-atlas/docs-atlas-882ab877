Every request to the Stophy API must include a valid API key. Requests without a key, or with an invalid key, are rejected before any processing occurs — and no credits are deducted.

## Passing your API key

Include your API key in one of two request headers. Both are accepted and behave identically:

| Header | Example |
|--------|---------|
| `x-api-key` | `x-api-key: sk_live_abc123` |
| `api-x-key` | `api-x-key: sk_live_abc123` |

The server checks for `api-x-key` first. If that header is absent, it falls back to `x-api-key`.

<Warning>
  Your API key is a secret. Never include it in client-side JavaScript, commit it to version control, or share it publicly. Use environment variables or a secrets manager to store it.
</Warning>

## Examples

<CodeGroup>

```bash curl
curl -X POST https://your-stophy-host/v1/video/transcript \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{"videoUrl": "https://www.youtube.com/watch?v=dQw4w9WgXcQ"}'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-stophy-host/v1/video/transcript', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': process.env.STOPHY_API_KEY,
  },
  body: JSON.stringify({
    videoUrl: 'https://www.youtube.com/watch?v=dQw4w9WgXcQ',
  }),
});
```

```python Python (requests)
import os
import requests

response = requests.post(
    'https://your-stophy-host/v1/video/transcript',
    headers={'x-api-key': os.environ['STOPHY_API_KEY']},
    json={'videoUrl': 'https://www.youtube.com/watch?v=dQw4w9WgXcQ'},
)
```

</CodeGroup>

## Error responses

### Missing API key

If neither `api-x-key` nor `x-api-key` is present in the request, the server returns:

```json
{
  "statusCode": 401,
  "message": "Missing api-x-key header"
}
```

**HTTP status:** `401 Unauthorized`

### Invalid API key

If the key is present but does not match any record in the database:

```json
{
  "statusCode": 401,
  "message": "Invalid API key"
}
```

**HTTP status:** `401 Unauthorized`

<Note>
  In rare cases, a valid API key may be linked to a user account that no longer exists. This returns a 404 response:

  ```json
  {
    "statusCode": 404,
    "message": "User not found for API key"
  }
  ```

  If you receive this error, contact support — it indicates a data consistency issue on the server side.
</Note>

## How authentication works

When a request arrives, the server:

1. Reads the `api-x-key` or `x-api-key` header value
2. Looks up the key in the `api_key` table
3. Resolves the associated user from the `user` table
4. Checks the user's credit balance before proceeding

Authentication and credit checks happen before any YouTube data is fetched, so failed auth requests never consume credits.
